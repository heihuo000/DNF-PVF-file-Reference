## 📝 编写技巧和最佳实践

### 1. 标签顺序建议
```
// 推荐的标签顺序
1. 基本信息 (name, grade, rarity, etc.)
2. 使用限制 (usable job, minimum level, etc.)
3. 装备类型 (equipment type, sub type)
4. 装备属性 (attack, defense, HP, MP, etc.)
5. 特殊效果 (skill, critical, damage rate, etc.)
6. 套装效果 (set name, set item, set ability)
7. 视觉音效 (icon, animation, sound, etc.)
8. 其他属性 (weight, price, durability, etc.)
```

### 2. 数值平衡原则
- 装备等级越高，属性数值越大
- 稀有度影响属性的丰富程度
- 套装效果应该有明显的吸引力
- 特殊效果不要过于强大

### 3. 兼容性考虑
- 职业限制要合理设置
- 等级限制要符合游戏进度
- 套装物品ID要保持一致
- 文件路径要确保存在

### 4. 注释使用
```
// 使用注释分隔不同功能区域
// ==================== 基本信息 ====================
// ==================== 装备属性 ====================
// ==================== 特殊效果 ====================
```
